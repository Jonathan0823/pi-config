# Project instructions

- Create or update `tasks/<name>.md` before implementation.
- Keep `tasks/` as the source of truth.
- Ask clarifying questions when the request is vague.
- Make small, testable changes.
- Keep the task file current after compaction or context resets.
- Run safe validation (lint/typecheck/tests) before handoff.
- Ask before risky commands or secret/path access.
